## My Approach

I treat each competition as an end-to-end ML project:

### 1. Problem Framing
- Understand evaluation metric and constraints
- Identify potential sources of leakage
- Form hypotheses before modeling

### 2. Baseline & Validation
- Start with simple models
- Align cross-validation strategy with competition metric
- Track performance improvements methodically

### 3. Iterative Feature Engineering
- Create domain-informed features
- Test interaction terms and non-linear relationships
- Evaluate model impact through cross-validation

### 4. Model Development
- Compare tree-based models, linear models, and neural networks when appropriate
- Apply hyperparameter tuning (Optuna, early stopping)
- Use ensembling only when justified by validation performance

### 5. Reflection
- Analyze failure cases
- Identify overfitting risks
- Document lessons learned
